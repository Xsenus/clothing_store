# YooMoney Setup

Интеграция YooMoney в проект поддерживает:

- выставление счета при создании заказа;
- прием HTTP-уведомлений от YooMoney;
- ручную и фоновую сверку оплаты через `operation-history`;
- повторную выдачу формы оплаты из профиля пользователя;
- ручную проверку оплаты из админки.

## Что заполнить в интеграциях

В админке откройте `Настройки -> Интеграции -> YooMoney` и заполните:

- `Включить оплату через YooMoney`
- `Номер кошелька`
- `Секрет для уведомлений`
- `Access token`
- `Префикс метки`
- `Срок жизни счета, минут`
- `Разрешить банковские карты`
- `Разрешить оплату кошельком YooMoney`

Минимально для полноценной работы нужны:

- номер кошелька;
- секрет уведомлений;
- access token c доступом к истории операций.

## URL уведомлений

В кабинете YooMoney настройте HTTP-уведомления на backend endpoint:

```text
/api/integrations/yoomoney/notifications
```

В админке этот URL показывается уже в абсолютном виде, готовым для копирования.

## Что делает access token

`Access token` нужен не для выдачи формы оплаты, а для проверки операций:

- ручная кнопка `Проверить оплату` у пользователя;
- ручная проверка в админке;
- фоновый монитор pending-платежей.

Без него webhook-уведомления продолжат работать, но ручная/фоновая сверка будет недоступна.

## Поток оплаты

1. Пользователь оформляет заказ с методом `ЮMoney`.
2. Backend создает запись `order_payments` и отдает HTML form payload для YooMoney Quickpay.
3. Frontend отправляет пользователя в форму YooMoney.
4. После оплаты YooMoney шлет уведомление на наш webhook.
5. Если webhook задержался, система может подтянуть оплату через `operation-history`.

## Полезные замечания

- `label` генерируется автоматически и используется как основной идентификатор платежа.
- Для карт и кошелька backend учитывает комиссию и сохраняет ожидаемую сумму зачисления.
- Если заказ переведен в терминальный статус, активный pending-счет отменяется внутри системы.
