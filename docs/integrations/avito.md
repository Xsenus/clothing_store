# Avito

## Официальные материалы

- Портал разработчика: <https://developers.avito.ru/>
- Каталог API: <https://developers.avito.ru/api-catalog>
- OAuth token endpoint: <https://api.avito.ru/token>

## Что реализовано в проекте

- Админская проверка доступа к официальному OAuth endpoint Avito.
- Подготовка конфигурации под возможное подключение Avito как провайдера доставки.
- Прозрачная обработка ограничения: если публичного storefront delivery API нет, система не притворяется, что интеграция завершена.

Ключевые файлы:

- `backend/Store.Api/Services/AvitoDeliveryService.cs`
- `src/components/admin/AdminNewIntegrationsTabs.tsx`
- `src/pages/admin.tsx`

## Настройки

- `delivery_avito_enabled`
- `delivery_avito_client_id`
- `delivery_avito_client_secret`
- `delivery_avito_scope`
- `delivery_avito_warehouse_address`
- `delivery_avito_notes`

## Что важно понимать

На дату реализации публичная официальная документация Avito, доступная через портал разработчика, подтверждает OAuth и каталог API, но не дает открытого внешнего delivery API для типового интернет-магазина, который можно безопасно встроить в ваш checkout на тех же условиях, что СДЭК или Почта России.

Поэтому в проекте сделано следующее:

- есть проверка официального OAuth-контура;
- есть готовое место в настройках и сервисном слое;
- нет ложной имитации расчета тарифов или создания отправлений без подтвержденной публичной спецификации.

## Тестовый контур

1. Включить `delivery_avito_enabled=true`.
2. Заполнить `client_id` и `client_secret`.
3. В админке открыть вкладку `Avito`.
4. Выполнить OAuth test.

Что тестируется:

- доступность `https://api.avito.ru/token`;
- корректность клиентских учетных данных;
- готовность контура к дальнейшей интеграции.

## Ограничения текущей реализации

- Расчет стоимости доставки Avito в checkout не включен.
- Список ПВЗ/постаматов Avito не включен.
- Создание отправления и отслеживание статуса не включены.

Причина не техническая, а продуктовая и документарная: без официальной публичной спецификации для внешнего магазина такую интеграцию нельзя внедрять надежно и безопасно.

## Что потребуется для продолжения

- Подтвержденный публичный или партнерский delivery API Avito.
- Официальная схема создания отправления, расчета тарифа, списка точек и статусов.
- После появления такой спецификации можно быстро расширить текущий сервисный слой и подключить Avito в общий агрегатор доставки.
