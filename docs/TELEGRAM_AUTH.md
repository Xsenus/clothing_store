# Авторизация через Telegram

Документ описывает обновлённый сценарий входа через Telegram-бота (по deep-link), где для нового пользователя запрашивается номер телефона, а для уже авторизованного — вход завершается сразу.

## 1) Полный поток авторизации

1. Фронтенд (`/auth`) читает публичные настройки через `GET /settings/public`.
2. Кнопка «Войти через Telegram» показывается только если:
   - `telegram_login_enabled=true`,
   - задан `telegram_bot_username`.
3. При нажатии кнопки фронтенд вызывает `POST /auth/telegram/start`.
4. Бэкенд создаёт `telegram_auth_requests` со `state` и возвращает ссылку:
   - `https://t.me/<bot_username>?start=login_<state>`.
5. Пользователь переходит в бота по ссылке.
6. Бот обрабатывает `/start login_<state>`:
   - если пользователь уже есть в системе (`telegram_<id>@telegram.local`) — помечает запрос как `completed`;
   - если пользователь новый — переводит запрос в `awaiting_phone` и просит отправить контакт (кнопка `request_contact=true`).
7. Новый пользователь отправляет контакт (номер телефона), бот:
   - создаёт `users` + `profiles`,
   - заполняет `profiles.phone`,
   - выставляет `profiles.phone_verified=true`,
   - завершает запрос (`completed`).
8. Фронтенд опрашивает `GET /auth/telegram/status/{state}`.
9. Когда запрос завершён, API создаёт `sessions` + `refresh_sessions`, возвращает токены, и фронтенд открывает `/profile`.

---

## 2) Состояния `telegram_auth_requests`

- `pending` — запрос создан, пользователь ещё не открыл бота.
- `awaiting_phone` — пользователь в боте, ждём контакт.
- `completed` — бот завершил идентификацию, сайт может забрать токены.
- `consumed` — токены уже выданы, повторный обмен запрещён.
- `expired` — запрос протух по TTL.

TTL запроса: 10 минут.

---

## 3) Что хранится в БД

### `profiles`
- добавлено поле `phone_verified` (bool).

### `telegram_auth_requests`
- `state` (уникальный),
- `bot_id`,
- `telegram_user_id`, `chat_id`,
- `user_id`, `phone_number`,
- `status`, `created_at`, `expires_at`, `completed_at`, `consumed_at`.

---

## 4) Поведение для двух сценариев

### 4.1 Пользователь уже входил ранее
- Нажимает кнопку на сайте.
- Переходит в бота.
- Бот узнаёт Telegram ID и сразу завершает запрос.
- Сайт автоматически авторизует и открывает профиль.

### 4.2 Пользователь заходит впервые
- Нажимает кнопку на сайте.
- Переходит в бота.
- Бот просит отправить номер телефона.
- После отправки контакта создаётся аккаунт, телефон становится подтверждённым.
- Сайт автоматически авторизует и открывает профиль.
